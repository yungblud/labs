# 미적분 이론 노트 — 정의에서 공식까지 직접 유도

> 스펙: [`../specs/calculus-by-hand.md`](../specs/calculus-by-hand.md)
> 방식: **순수 이론(손유도) 중심.** 극한·수렴처럼 눈으로 보면 확 와닿는 데서만 몇 줄 코드.
> 톤: **12살에게 설명하듯** — 유도는 한 줄씩, 왜 그 줄이 나오는지 말로 붙인다. 건너뛰는 줄 없음.
>
> 대상 함수(손검산 가능한 최소셋): `f(x)=x²`, `f(x)=x³`, `g(x)=2x`

---

## 목차

- **파트 1 — 극한: 모든 것의 뿌리**
  - 순간의 기울기는 왜 `0/0` 에 부딪히나
  - `h → 0` 의 진짜 의미 ("대입"이 아니라 "향하는 곳")
- **파트 2 — 미분: 정의에서 공식 유도**
  - 미분계수의 정의
  - `x²` 를 한 줄씩 전개 → `2x`
  - `x³` 자기 유도 → `3x²`
- **파트 3 — 미분 규칙이 왜 성립하나**
  - 거듭제곱 규칙 `xⁿ → n·xⁿ⁻¹`
  - 상수배·합 규칙
- **파트 4 — 적분: 넓이의 극한**
  - 리만 합과 `n → ∞`
  - 부정적분과 `+C` 의 의미
  - `∫₀³ 2x = 9` 삼각형 손검산
- **파트 5 — 기본정리: 두 축의 연결**
  - "넓이를 미분하면 원래 함수"
  - `2x ↔ x²` 왕복

## 파트 1 — 극한: 모든 것의 뿌리

### 1-1. "순간의 기울기"를 재려는 순간, 벽에 부딪힌다

기울기가 뭐였지? **세로로 변한 만큼 ÷ 가로로 변한 만큼** 이야. 두 점이 있어야 잴 수 있어.

```
기울기 = (세로 변화) / (가로 변화)
```

그런데 우리가 알고 싶은 건 **"딱 한 점에서의 기울기"** 야. 자동차 계기판이 "지금 이 순간"의
속도를 보여주는 것처럼. 근데 점이 하나뿐이면? 가로 변화도 0, 세로 변화도 0.

```
기울기 = 0 / 0     ← ???
```

`0/0` 은 답이 없어. (3 나누기 0 은 "불가능"이고, 0 나누기 0 은 "뭐든 될 수 있음" 이라 더 나빠.)
**한 점에서 기울기를 바로 재려 하면 반드시 이 벽에 부딪힌다.** 이게 미적분이 풀어야 했던 문제야.

### 1-2. 극한의 아이디어 — "닿지 말고, 다가가라"

똑똑한 우회법이 있어. 한 점에 **바로 앉지 말고**, 아주 가까운 두 번째 점을 잡아서 기울기를 재.
그리고 그 두 번째 점을 **점점 더 가까이** 옮기면서 값이 어디로 향하는지 봐.

- 두 점 사이 간격을 `h` 라고 부르자 (h = 가로로 벌린 거리).
- `h` 를 1 → 0.1 → 0.01 → ... 이렇게 **0 쪽으로 줄여** 가.
- `h` 를 진짜 0 으로 만들면 다시 `0/0` 벽이니까, **0 에 닿지는 않고 무한히 다가가**.

이렇게 "닿지 않고 다가갈 때 값이 향하는 곳"을 **극한(limit)** 이라고 불러. 기호로:

```
lim_{h→0}  (어떤 식)
```

`h→0` 은 **"h 에 0 을 대입해라"가 아니야.** "h 를 0 에 한없이 가까이 보낼 때, 이 식이 어떤 값에
다가가는가?" 라는 뜻이야. 이 한 끗 차이가 미적분 전체를 떠받쳐.

### 1-3. 눈으로 확인 (극한이라 몇 줄 코드 허용)

`f(x)=x²` 를 `x=2` 에서 재보자. 두 점 `2` 와 `2+h` 사이 기울기:

```
slope(h) = ( f(2+h) − f(2) ) / h = ( (2+h)² − 4 ) / h
```

`h` 를 줄여가며 실제로 돌린 결과 (`limit_check.py`):

```
           h |           slope(h) |         |오차|
------------------------------------------------
       1e+00 |     5.000000000000 |     1.00e+00
       1e-01 |     4.100000000000 |     1.00e-01
       1e-02 |     4.010000000000 |     1.00e-02
       1e-03 |     4.001000000000 |     1.00e-03
       1e-04 |     4.000100000008 |     1.00e-04
       1e-05 |     4.000010000027 |     1.00e-05
       1e-06 |     4.000001000648 |     1.00e-06
       1e-07 |     4.000000091153 |     9.12e-08
       1e-08 |     3.999999975690 |     2.43e-08
       ...
       1e-12 |     4.000355602329 |     3.56e-04   ← 다시 나빠진다!
       1e-15 |     3.552713678801 |     4.47e-01   ← 더 나빠진다!
```

**관찰 1 (핵심):** `h` 를 줄일수록 slope 가 **4** 에 착착 붙는다. 이게 극한 `lim_{h→0} slope(h) = 4`.

**관찰 2 (정직하게):** 근데 `h` 가 너무 작아지면(1e-12 이하) 값이 되레 **흔들린다.** 왜?
컴퓨터는 실수를 유한한 자릿수로만 저장해서, `4` 와 `4.000000...001` 같이 **거의 같은 두 수를 빼면**
믿을 만한 자릿수가 다 날아가(자릿수 소실). 즉 **"진짜 극한"은 수학의 이상**이고, 컴퓨터의 근사는
어느 지점부턴 오히려 나빠져. 극한은 계산이 아니라 **논리로** 도달해야 함을 이 표가 보여준다.

→ 그래서 다음 파트에서 `h` 를 대입하지 않고 **식으로 정리해** 정확히 4 를 뽑는다.

## 파트 2 — 미분: 정의에서 공식 유도

> ### 📦 개념 박스 — 미분을 오해 안 하려면 (먼저 읽기)
>
> **① 함수는 "기계"다.** `f(x)=x¹⁰⁰` 은 x 를 넣으면 값을 뱉는 규칙.
> - **지수 100 은 고정** (기계의 모양). **변하는 건 입력 x 하나뿐.**
> - "숫자가 변한다"의 그 숫자는 지수가 아니라 **입력 x**.
>
> **② 가로 = 입력 x, 세로 = 출력 y(=f(x)).**
> - 가로변화 = 입력을 민 거리 = `h`  (함수 상관없이 항상 h)
> - 세로변화 = 그때 출력이 뛴 양 = `f(x+h) − f(x)`  (이게 함수마다 달라지는 부분)
>
> **③ 미분은 "값"이 아니라 "변화 속도"를 구한다.** 두 질문은 다르다:
>
> | 질문 | 답을 주는 것 |
> | --- | --- |
> | "지금 값이 얼마?" | 원래 수식 `f(x) = x²` |
> | "지금 얼마나 빨리 변해?" | 미분한 수식 `f'(x) = 2x` |
>
> **④ 미분은 새 기계를 하나 만든다.** 원래 기계 `f(x)=x²` 를 재료로, **변화 속도를 주는
> 새 기계** `f'(x)=2x` 가 나온다. `f'` 도 x 를 넣으면 "그 지점의 기울기"를 뱉는 기계.
>
> → 즉 미분이 알고 싶은 건 **"입력이 조금 변할 때 출력이 얼마나 빨리 따라 변하나"**,
> 그것도 **딱 그 순간(한 점)** 에서. 그 "순간"으로 좁히는 도구가 `lim_{h→0}`.

### 2-1. 미분계수의 정의

파트 1 의 "두 점 기울기의 극한"을 그대로 식으로 굳힌 게 **미분** 이야. `x` 에서의 순간 기울기를
`f'(x)` ("에프 프라임") 라 쓰고, 이렇게 정의해:

```
f'(x) = lim_{h→0}  [ f(x+h) − f(x) ] / h
```

- `f(x+h)` = 살짝 옆으로(h 만큼) 간 점의 높이
- `f(x)`   = 원래 점의 높이
- `h`      = 두 점의 가로 간격
- `lim_{h→0}` = 그 간격을 0 으로 좁혔을 때 향하는 값

이건 새로운 게 아니라 **파트 1 에서 손으로 한 걸 기호로 옮긴 것뿐.** 코드는 `h` 를 대입했지만,
이제 우리는 **대입하지 않고 식을 정리**해서 `0/0` 벽을 논리로 통과할 거야.

### 2-2. `x²` 를 한 줄씩 유도 → `2x`

`f(x)=x²` 를 정의에 넣고, **한 줄씩 왜 그렇게 되는지** 붙이며 간다.

```
f'(x) = lim_{h→0} [ (x+h)² − x² ] / h
```

**① 분자 (x+h)² 를 전개.**  (x+h)² = x² + 2xh + h²  이니까:
```
        = lim_{h→0} [ x² + 2xh + h² − x² ] / h
```

**② x² − x² 는 사라진다.**  ← 이게 마법의 순간. 방해하던 x² 가 상쇄돼:
```
        = lim_{h→0} [ 2xh + h² ] / h
```

**③ 분자·분모를 h 로 나눈다.**  분자의 모든 항에 h 가 있으니 약분돼:
```
        = lim_{h→0} [ 2x + h ]
```

**④ 이제 h→0 을 안전하게 보낸다.**  여기엔 더 이상 `0/0` 이 없어! `2x + h` 에서 h 만 0 으로
보내면 뒤 항이 조용히 사라져:
```
f'(x) = 2x
```

**끝.** `x²` 의 도함수는 `2x`. 검산: `x=2` 넣으면 `f'(2) = 2·2 = 4` — 파트 1 코드가 붙어가던
바로 그 **4**. 노가다(코드)와 논리(유도)가 같은 답에서 만났다.

> **왜 극한이 필요했나 다시 보기:** ③단계까지 오면 식이 `2x+h` 라 h=0 을 넣어도 멀쩡해.
> 하지만 ①②③ 을 하기 **전**엔 `0/0` 이었어. 극한은 "0/0 을 식으로 풀어낼 시간을 벌어주는" 장치다.

### 2-3. `x³` 자기 유도 → `3x²`

같은 절차를 `f(x)=x³` 로 혼자 해보면 규칙이 보인다. 필요한 전개: (x+h)³ = x³ + 3x²h + 3xh² + h³.

```
f'(x) = lim_{h→0} [ (x+h)³ − x³ ] / h
      = lim_{h→0} [ x³ + 3x²h + 3xh² + h³ − x³ ] / h     ← 전개
      = lim_{h→0} [ 3x²h + 3xh² + h³ ] / h               ← x³ 상쇄
      = lim_{h→0} [ 3x² + 3xh + h² ]                      ← h 로 약분
      = 3x²                                                ← h→0: h 붙은 항 소멸
```

`x³ → 3x²`. `x² → 2x` 와 나란히 놓으면 패턴이 보이지?

```
x²  →  2x       (2 = 지수, 1 = 지수−1)
x³  →  3x²      (3 = 지수, 2 = 지수−1)
```

**"지수를 앞으로 내리고, 지수는 1 줄인다."** 이 규칙이 왜 항상 성립하는지는 파트 3 에서.

## 파트 3 — 미분 규칙이 왜 성립하나

파트 2 에서 `x² → 2x`, `x³ → 3x²` 를 **일일이 손유도**했어. 그런데 매번 이렇게 할 순 없잖아.
`x¹⁰⁰` 은? 여기선 규칙 세 개가 **왜** 성립하는지 정의에서 뽑아낸다. "외운다"가 아니라 "떨어진다".

### 3-0. 먼저 필요한 재료: (x+h)ⁿ 의 앞 두 항

`(x+h)ⁿ` 을 다 전개할 필요 없어. 미분에서 **살아남는 건 앞 두 항뿐**이거든. 전개하면 이렇게 생겼어:

```
(x+h)ⁿ = xⁿ  +  n·xⁿ⁻¹·h  +  (h² 이상이 붙은 나머지 항들)
```

- 첫 항 `xⁿ` : h 를 하나도 안 고른 것 (x 만 n 번 곱함).
- 둘째 항 `n·xⁿ⁻¹·h` : h 를 딱 하나 고른 것. n 개 자리 중 하나에서 h 를 고르므로 **n 가지** → 계수 n.
- 나머지 : h 를 둘 이상 고른 것 → 전부 `h²`, `h³`, ... 이 붙음.

검산 (파트 2 와 일치):
```
(x+h)² = x² + 2xh + h²          → 앞 두 항: x² + 2xh      (n=2)
(x+h)³ = x³ + 3x²h + 3xh² + h³  → 앞 두 항: x³ + 3x²h     (n=3)
```

### 3-1. 거듭제곱 규칙:  xⁿ → n·xⁿ⁻¹

`f(x)=xⁿ` 을 정의에 넣는다. 파트 2 와 **똑같은 4 박자**야.

```
f'(x) = lim_{h→0} [ (x+h)ⁿ − xⁿ ] / h
```

**① (x+h)ⁿ 전개** (위 재료 사용):
```
      = lim_{h→0} [ xⁿ + n·xⁿ⁻¹·h + (h²이상 항들) − xⁿ ] / h
```

**② xⁿ 상쇄** (첫 항이 뒤의 −xⁿ 과 사라짐):
```
      = lim_{h→0} [ n·xⁿ⁻¹·h + (h²이상 항들) ] / h
```

**③ h 로 약분** (남은 항엔 전부 h 가 최소 하나씩 있음):
```
      = lim_{h→0} [ n·xⁿ⁻¹ + (h¹이상 항들) ]
```
여기서 핵심: `h²` 이던 항은 약분 후 `h¹` 이 되고, `h³` 은 `h²` 가 되고... 즉 **첫 항 빼고 전부 아직
h 를 달고 있다.**

**④ h→0** (h 를 단 항은 전부 사라지고, h 없는 첫 항만 생존):
```
f'(x) = n·xⁿ⁻¹
```

**증명 끝.** 이게 "지수를 앞으로 내리고 지수를 1 줄인다"의 정체야. 왜 하필 그렇게 되냐면 —
**h 를 정확히 하나 고른 항**만 약분 뒤에 살아남기 때문. 그 항의 계수가 n(고르는 자리가 n 개),
남는 x 는 n−1 개(`xⁿ⁻¹`). 그래서 `n·xⁿ⁻¹`.

검산:
```
n=2:  x² → 2·x¹  = 2x       ✅ (파트 2 와 일치)
n=3:  x³ → 3·x²  = 3x²      ✅
n=100: x¹⁰⁰ → 100·x⁹⁹       (이젠 손유도 없이 즉시)
```

### 3-2. 상수배 규칙:  c·f(x) → c·f'(x)

함수 앞에 붙은 상수 `c` 는 미분 밖으로 그냥 빠져나온다. 왜? 정의에 넣으면 c 가 **공통인수로 묶여**:

```
[c·f]'(x) = lim_{h→0} [ c·f(x+h) − c·f(x) ] / h
          = lim_{h→0}  c · [ f(x+h) − f(x) ] / h      ← c 를 묶어냄
          = c · lim_{h→0} [ f(x+h) − f(x) ] / h        ← 상수는 극한 밖으로
          = c · f'(x)
```

예: `g(x)=2x` → 상수 2 를 빼면 `2·(x)'`. 그리고 `(x)' = 1·x⁰ = 1` (거듭제곱 규칙, n=1). 따라서
```
g'(x) = 2 · 1 = 2
```
직선 `y=2x` 의 기울기는 어디서나 2 — 당연하지, 직선이니까. 규칙이 상식과 맞는다.

### 3-3. 합 규칙:  [f + g]' → f' + g'

더해진 함수는 **따로따로 미분해서 더하면** 된다. 정의에서 분자를 재배열하면 두 덩어리로 쪼개져:

```
[f+g]'(x) = lim_{h→0} [ (f(x+h)+g(x+h)) − (f(x)+g(x)) ] / h
          = lim_{h→0} [ (f(x+h)−f(x)) + (g(x+h)−g(x)) ] / h   ← f끼리, g끼리 모음
          = lim_{h→0} (f(x+h)−f(x))/h  +  lim_{h→0} (g(x+h)−g(x))/h   ← 극한은 합으로 쪼갬
          = f'(x) + g'(x)
```

### 3-4. 세 규칙을 합치면 — 다항식은 이제 암산

이 셋(거듭제곱 + 상수배 + 합)이면 **어떤 다항식이든** 손유도 없이 바로 미분된다. 예:

```
f(x) = 3x² + 5x + 7

  3x² → 3·(2x) = 6x       (상수배 + 거듭제곱)
  5x  → 5·(1)  = 5        (상수배 + 거듭제곱, x=x¹)
  7   → 0                 (상수는 안 변함 → 기울기 0; x⁰ 규칙으로도 0)

f'(x) = 6x + 5
```

> **상수의 미분이 0 인 이유:** `f(x)=7` 은 높이가 늘 7 인 **평평한 수평선**. 아무리 x 가 변해도
> 높이는 안 변하니 기울기는 0. 이 사실은 파트 4(적분)의 `+C` 에서 다시 등장한다.

## 파트 4 — 적분: 넓이의 극한

미분이 "쪼개서 순간을 봤다"면, 적분은 반대로 **"잘게 쪼갠 걸 전부 더해 전체를 본다".**
그런데 놀랍게도 **엔진은 똑같이 `lim`** 이야 — 미분은 `lim_{h→0}`, 적분은 `lim_{n→∞}`.

### 4-1. 왜 "넓이"가 목표인가

곡선 아래 넓이가 왜 중요하냐면, 그게 **"쌓인 총량"** 이거든.
- 속도 그래프 아래 넓이 = 이동한 **총 거리**
- 시간당 물 붓는 양 그래프 아래 넓이 = 채운 **총 물의 양**

문제는 곡선 아래는 사각형이 아니라 **넓이 공식이 없다**는 것. 그래서 미분에서 `0/0` 벽을 만났듯,
여기선 **"반듯하지 않은 도형의 넓이"** 벽을 만난다.

### 4-2. 리만 합 — 막대기로 채워서 더한다

우회법은 미분과 똑 닮았어. 정확히 못 재니까, **잘게 쪼개서 근사**하고 극한으로 넘어간다.

`g(x)=2x` 의 `x=0..3` 아래 넓이를 구해보자.

1. 구간 `[0,3]` 을 **막대기 n 개**로 나눈다. 각 막대 폭 `Δx = 3/n`.
2. 각 막대의 넓이 ≈ **(높이 × 폭)** = `g(xᵢ) · Δx`. (곡선을 그 폭에선 "평평하다" 치는 것)
3. 전부 더한다:
```
area(n) = g(x₁)·Δx + g(x₂)·Δx + ... + g(xₙ)·Δx = Σᵢ g(xᵢ)·Δx
```
4. 막대를 무한히 잘게(`n→∞`) 만들면 근사가 참값이 된다:
```
넓이 = lim_{n→∞} Σᵢ g(xᵢ)·Δx
```

이게 **정적분**이고, 기호로 이렇게 쓴다:
```
∫₀³ 2x dx
```
- `∫` = 길게 늘린 S(sum). "다 더해라".
- `dx` = 무한히 얇아진 막대 폭 (`Δx` 가 0 으로 간 것).
- `0`, `3` = 더하는 구간의 시작과 끝.

즉 `∫₀³ 2x dx` 는 통째로 읽으면 **"2x 를 0 부터 3 까지, 얇은 막대로 쪼개 전부 더한 것"**.

### 4-3. 눈으로 확인 (극한이라 몇 줄 허용)

막대 높이를 막대의 **왼끝 / 오른끝 / 중점** 중 어디서 잴지에 따라 근사가 조금 달라. 셋 다 돌려봤어
(`area_check.py`):

```
       n |         left |        right |          mid
----------------------------------------------------
      10 |     8.100000 |     9.900000 |     9.000000
     100 |     8.910000 |     9.090000 |     9.000000
    1000 |     8.991000 |     9.009000 |     9.000000
   10000 |     8.999100 |     9.000900 |     9.000000

참값 → 9.0
```

**관찰 1:** 왼끝은 **아래로**(작게), 오른끝은 **위로**(크게) 빗나가지만, `n` 이 커질수록 둘 다 **9** 로
조여든다. 이게 `lim_{n→∞} = 9`. 미분의 수렴 표와 판박이지 — 이번엔 `h→0` 대신 `n→∞`.

**관찰 2:** **중점(mid)은 n=10 에서도 이미 정확히 9.** 왜? `2x` 는 **직선**이라, 막대 안에서 왼쪽에서
모자란 만큼 오른쪽에서 남아 **정확히 상쇄**돼(중점이 평균 높이라서). 함수가 직선이면 중점 리만 합은
근사가 아니라 정답. 곡선(`x²` 등)이면 중점도 근사가 되고 `n→∞` 가 필요해진다.

### 4-4. 손검산 — 삼각형으로

`g(x)=2x`, `x=0..3` 아래는 사실 **삼각형**이야:
- `x=0` 에서 높이 `0`, `x=3` 에서 높이 `2·3=6`. 밑변 3, 높이 6 인 직각삼각형.
```
삼각형 넓이 = ½ · 밑변 · 높이 = ½ · 3 · 6 = 9
```
리만 합이 붙어간 **9** 와 정확히 일치. 초등학교 넓이 공식과 적분이 같은 답에서 만났다.

### 4-5. 부정적분과 `+C` — 미분을 거꾸로

매번 막대기 수천 개를 더할 순 없잖아. 다행히 **적분은 미분의 역연산**이라, 미분 규칙을 거꾸로
돌리면 공식이 나와. 미분에서 `x² → 2x` 였으니, 거꾸로:
```
2x 를 적분하면 → x²
```
거듭제곱 규칙도 거꾸로 (미분은 "내리고 줄이고", 적분은 **"올리고 나누고"**):
```
∫ xⁿ dx = xⁿ⁺¹ / (n+1) + C      (지수 1 올리고, 그 수로 나눔)
```
검산: `∫ 2x dx = 2 · x²/2 = x²` ✅

**`+C` 는 왜 붙나?** 파트 3-4 의 복선이 여기서 회수돼. `x²` 도, `x²+5` 도, `x²+100` 도 **미분하면 전부
`2x`** 야 (상수는 미분하면 0 으로 사라지니까). 그러니 `2x` 를 거꾸로 되돌릴 땐 "원래 상수가 얼마였는지
알 수 없다" → **"어떤 상수 C 가 있었을 수도 있다"** 는 뜻으로 `+C` 를 붙인다.

### 4-6. 정적분 = 부정적분에 끝값 넣고 빼기

넓이(정적분)를 구할 땐, 부정적분을 구한 뒤 **끝값 − 시작값**:
```
∫₀³ 2x dx = [ x² ]₀³ = (3²) − (0²) = 9 − 0 = 9
```
`+C` 는 어차피 빼면서 사라지니(`(9+C) − (0+C) = 9`) 정적분에선 신경 안 써도 된다. 그리고 이 **9** 는
4-3 의 리만 합, 4-4 의 삼각형과 **완전히 같은 값**. 세 방법(막대 합 / 삼각형 / 부정적분)이 한 점에서 만났다.

> **미분과의 대칭 한눈에:**
> ```
> 미분:  lim_{h→0}  잘게 쪼개 순간의 기울기   →  "내리고 줄이고"   x² → 2x
> 적분:  lim_{n→∞}  잘게 쪼갠 조각을 다 더함   →  "올리고 나누고"   2x → x²
> ```
> 두 화살표가 서로 반대인 게 눈에 보이지? 이 "서로 되돌린다"를 정식으로 묶는 게 파트 5.

## 파트 5 — 기본정리: 두 축의 연결

파트 2~3 은 미분, 파트 4 는 적분을 따로 세웠어. 이제 둘이 **같은 동전의 양면**임을 못박는다.
이게 미적분에서 제일 유명한 문장 — **미적분의 기본정리(FTC)**.

### 5-1. 한 문장

> **"넓이(쌓임)를 다시 미분하면, 원래 함수로 돌아온다."**

기호로: `2x` 를 적분해 `x²` (넓이 함수)를 얻고, 그 `x²` 를 미분하면 다시 `2x`.
```
2x  ──적분(쌓기)──▶  x²  ──미분(순간변화)──▶  2x
```
갔다가 제자리. 미분과 적분은 **곱셈↔나눗셈처럼 서로를 취소하는 한 쌍.**

### 5-2. 왜 그런가 — "넓이가 자라는 속도"로 이해

`A(x)` 를 `g` 아래 넓이가 `0` 부터 `x` 까지 쌓인 양이라 하자. `x` 를 아주 조금(`h`) 늘리면 넓이가
얼마나 늘까? 새로 생기는 건 폭 `h`, 높이 `≈ g(x)` 인 **얇은 기둥 하나**:
```
넓이 증가분 A(x+h) − A(x)  ≈  g(x) · h
```
양변을 `h` 로 나누고 `h→0`:
```
A'(x) = lim_{h→0} [ A(x+h) − A(x) ] / h  =  g(x)
```
즉 **넓이 함수를 미분하면 원래 높이 함수 `g` 가 나온다.** 이게 기본정리의 심장 — 어렵지 않지?
"넓이가 자라는 속도 = 그 자리의 높이" 라는 당연한 관찰일 뿐이야.

### 5-3. 그래서 정적분이 "끝값 빼기"로 계산되는 이유

`A(x)` 가 `g` 의 부정적분(미분하면 `g` 가 되는 함수)이니까, `a`~`b` 넓이는:
```
∫ₐᵇ g(x) dx = A(b) − A(a)
```
파트 4-6 에서 `∫₀³ 2x = [x²]₀³ = 9` 로 "끝값 넣고 빼기" 했던 게 **바로 이 정리 덕분**이었다.
막대기 수천 개를 안 더해도 되는 이유 = 미분의 역(부정적분)만 찾으면 넓이가 공짜로 나오니까.

### 5-4. 활용 — 예측과 제어

기본정리가 실전에서 뜻하는 건 **"변화율 ↔ 누적을 자유롭게 왕복한다"** 는 것. 세상 문제는 대개 둘 중
하나만 주어지는데, 왕복이 되니 **한쪽만 알면 반대쪽을 계산으로 얻는다.**

```
아는 것        도구      얻는 것
────────────────────────────────────
위치를 안다    미분   →  속도·가속도
속도를 안다    적분   →  이동 거리
힘을 안다      적분   →  궤적(미래 위치)     ← 로켓이 어디 떨어지나
증가율을 안다  적분   →  미래 총량           ← 인구·전염병 예측
```

**두 개의 궁극 목적:**
- **예측(Predict):** 지금의 변화를 쌓아 **미래를 미리 안다** (적분). 로켓 발사 전 낙하점 계산.
- **제어(Control):** 원하는 미래를 정하고 **지금 뭘 할지 역산한다** (미분). "여기 떨어뜨리려면 어떤 힘?"

실제 로켓·자율주행·AI 학습은 이 둘을 **초당 수백 번 왕복**한다: 측정 → (미분) 어긋난 변화율 파악 →
(적분) 미래 재예측 → 목표와 비교해 조정 → 반복. **변하는 세계를 예측하고 원하는 결과로 밀어붙이는 것**,
그게 미적분을 300년간 과학의 언어로 만든 이유다.

### 5-5. 실험 전체를 관통한 하나의 뿌리

```
중학교 수학 = 멈춰있는 세계 (정적)      거리÷시간, 가로×세로, 직선 기울기
      │  lim (극한) 이라는 다리
      ▼
미적분     = 변하는 세계 (동적)         순간속도, 곡선 아래 넓이, 매 지점 기울기

  거리÷시간  ──lim h→0──▶  순간속도 (미분)   "내리고 줄이고"
  가로×세로  ──lim n→∞──▶  곡선 넓이 (적분)   "올리고 나누고"
                                              └ 기본정리로 서로 역 ┘
```

미분도 적분도, 뿌리는 **`lim` 하나**. "닿지 말고 무한히 다가가라"로 `0/0` 벽과 "공식 없는 넓이" 벽을
둘 다 통과했다. 미적분은 새 산수가 아니라, **정적인 산수를 `lim` 으로 감싸 동적 세계까지 연장한 것.**
