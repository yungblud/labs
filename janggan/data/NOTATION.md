# 장단 표기 규칙 (중립 중간표현)

이 폴더의 데이터는 v1/v2 어느 인코딩에도 치우치지 않은 **중립 IR**이다.
3단계에서 이 IR을 읽어 v1(flat)·v2(positional) 두 방식으로 토큰화한다.

> ⚠️ 여기 담긴 장단 패턴은 **연구용으로 양식화(stylized)한 근사치**다. 특정 유파·악보의
> 정확한 채보가 아니다. 논문 재현의 목적은 "장단다운 내부 구조"의 학습이지 복원 정확도가
> 아니므로(스펙 참조), 패턴이 **일관된 박 구조**만 가지면 실험 목적에 충분하다.

## 모델: 칸(박) × 위치 그리드

정간보의 "한 칸 = 한 박, 칸 안 위치 = 리듬 세분"을 그대로 옮긴다.

- 한 장단(cycle) = `beats` 개의 **박(칸)** 시퀀스
- 각 박 = 위치 셀로 등분 — **셀 개수는 `grid[beat]` 행 길이가 정본**
- 각 셀 = **구음 한 개** 또는 `null`(빈 셀)
- `null`(빈 셀) = 앞 타점의 지속 or 쉼 (정간보의 빈 정간과 동일 의미)

### 대칭 vs 비대칭 박

- **대칭** 장단(굿거리·자진모리·세마치)은 모든 박이 같은 등분 → `subdivisions`는 정수.
- **비대칭** 장단(엇모리 3+2+3+2)은 박마다 등분이 다름 → `subdivisions`는 **박별 리스트**(`[3,2,3,2]`).
  이 경우가 v1/v2 비교의 강한 시험대다: v2(위치식)는 칸 경계 `|`로 박별 등분을 자연히 담지만,
  v1(길이식)은 절대 길이로 풀어야 해서 비대칭 구조를 잃기 쉽다.

## 구음 어휘 (janggu 기준, 5종)

| 토큰 | 한글 | 타법 |
| --- | --- | --- |
| `deong` | 덩 | 양편(북편+채편) 동시 |
| `kung` | 쿵 | 북편(왼쪽, 저음) |
| `deok` | 덕 | 채편(오른쪽, 고음) |
| `gideok` | 기덕 | 채편 겹장식(그레이스) |
| `roll` | 더러러러 | 채편 굴림 — 편의상 한 셀 1토큰으로 표기(실연은 다타) |

> 어휘를 5종으로 좁게 유지한다 — vocabulary 크기가 v1/v2 비교의 핵심 변수라서
> 불필요한 토큰을 늘리지 않는다.

## JSON 스키마 (`patterns.json`)

```jsonc
{
  "meta": { "gu": ["deong","kung","deok","gideok","roll"], "note": "..." },
  "patterns": [
    {
      "name": "gutgeori",       // 장단 이름
      "variant": "basic",       // 변주 라벨
      "beats": 4,               // 박(칸) 수
      "subdivisions": 3,        // 박당 위치 등분
      "grid": [                 // beats × subdivisions
        ["deong", null, "gideok"],
        ["kung",  null, "roll"],
        ...
      ]
    }
  ]
}
```

## 데이터 저작 — shorthand 빌드 스크립트

`patterns.json`은 손저작이되 재현 가능하도록 `build_dataset.py`로 생성한다.
각 박을 한 글자 = 한 셀의 짧은 문자열로 적고(내가 고른 변형, 랜덤 아님) 스크립트가 그리드로 펼친다.

| 글자 | 구음 | · 글자 | = null(빈 셀) |
| --- | --- | --- | --- |
| `D` | deong(덩) | `K` | kung(쿵) |
| `T` | deok(덕) | `G` | gideok(기덕) |
| `R` | roll(더러러러) | `.` | null |

예: 굿거리 basic = `["D.G", "K.R", "D.G", "K.R"]` → 4박×3위치.
엇모리 basic = `["D.K", "TK", "D.G", "TK"]` → 박별 등분 3/2/3/2.

재생성: `python build_dataset.py` (→ `patterns.json` 덮어씀).

## 인코딩과의 연결 (3단계 예고)

같은 `grid` 하나에서 두 표현이 파생된다:

- **v2 (positional / 정간보식):** `| 0:deong 2:gideok | 0:kung 2:roll | ...`
  칸 경계 `|`, 셀 위치 index + 구음. duration을 위치로 흡수 → 어휘 작음.
- **v1 (flat / duration):** `deong(2) gideok(1) kung(2) roll(1) ...`
  각 타점을 지속 길이로. duration 토큰이 어휘로 들어가 → 어휘 큼.
