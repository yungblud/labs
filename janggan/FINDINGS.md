# 리서치 노트 — 장단 인코딩 실험

> 실험 스펙: [`../specs/janggan-encoding-research.md`](../specs/janggan-encoding-research.md)
> 출발 논문: [arXiv 2408.01096](https://arxiv.org/abs/2408.01096) *Six Dragons Fly Again*
> 성격: 논문 **결론의 재현**이 아니라 **연구 과정의 체험**. 결과가 null 이어도 그 이유를 이해하면 성공.

## 재현하려 한 주장

논문의 핵심: **"학습 데이터가 극도로 적을 때, 도메인 표기법(정간보)을 따른 인코딩을
설계하면 표준 트랜스포머가 잘 학습한다."** 특히 음길이를 별도 토큰이 아니라 **칸 안 위치**로
표현(JG-like)해 어휘를 줄인 것이 승부처였다.

이를 리듬 도메인으로 축소: 한국 장단을 정간보식 "칸 안 위치 = 리듬"으로 인코딩하고,
duration-기반(v1) vs position-기반(v2) 두 표현을 **동일 조건**에서 학습·비교한다.

## 셋업

- **데이터:** 굿거리·자진모리·세마치·엇모리(비대칭 3+2+3+2) 4종, 손저작 **34 시퀀스**. 박×위치 그리드 중립 IR.
- **인코딩:** v1 `deong d2 gideok d1 …`(박 경계 없음) / v2 `|3 p0 deong p2 gideok …`(칸 명시). 둘 다 왕복 무손실.
- **모델:** TinyGPT(2층·d64·4head). seed 1337·2000step·동일 하이퍼파라미터, **변수는 인코딩 하나뿐.**
- **측정:** 온도별 200개 생성 → parse·cycle(총길이 유효)·shape(박구조 유효, v2전용)·novel.

## 관찰 3단

### ① 어휘 축소가 재현되지 않았다
논문의 핵심 이점(위치 인코딩 → 어휘 감소)이 **역전**됐다: v1 어휘 8 < v2 어휘 10.
장단은 타점이 촘촘해 지속 길이가 **d1~d3**로 좁다 → 길이 토큰이 애초에 몇 개 안 된다.
논문의 어휘 폭발은 duration 범위가 넓어서(온음표·점음표…)였는데, **그 조건이 여기 없다.**

### ② 그래서 가설을 바꿨다 — 어휘 → 구조
v2 의 남는 이점은 어휘가 아니라 **박 구조가 토큰에 명시**된다는 것(`|3 |2`).
→ 수정 가설: **v1 은 박 경계를 못 담아 생성 시 박 구조를 더 자주 깰 것** (특히 비대칭 엇모리).

### ③ 그 가설도 기각됐다
| temp | v1 cycle | v2 cycle | v2 shape |
| --- | --- | --- | --- |
| 1.0 | 100 | 99.5 | 99.5 |
| 1.5 | 92.5 | 93.0 | 93.0 |
| 2.0 | **72** | **66.5** | 66.5 |

고온에서 오히려 **v1(72%) > v2(66.5%)**. v2 가 더 자주 깬다. 이유:
1. v2 는 `|N`·`pN` 토큰이 붙어 **시퀀스가 길다** → 고온 샘플링에서 토큰 하나 틀릴 기회가 많다(넓은 error surface).
2. 데이터 34개라 **둘 다 암기**(저온 novel 0%) → 저온에선 차이가 소멸.
3. duration 범위가 좁아 v1 이 애초에 불리하지 않다.

## 결론

**논문의 주장은 이 토이에서 재현되지 않았다. 그리고 그 이유가 배움이다.**

- 위치 인코딩의 이득은 **무조건적이지 않고 레짐 의존적**이다. 넓은 duration 범위 + 복잡한
  heterophony(논문의 궁중음악)에서 진짜 이득이 나고, 촘촘한 단선 리듬(우리 장단)에선 안 난다.
- v2 가 가진 **유일하게 확실한 이점**은 생성 품질이 아니라 **표현의 구조 checkability** —
  박 구조를 자동 보장·검증·제약할 수 있다(shape≈cycle). v1 은 shape 측정조차 불가.
  즉 "인코딩이 결과를 바꾼다"는 **성능이 아니라 검증가능성** 축에서 참이었다.
- 순진한 직관("정간보식 = 무조건 이득")이 깨졌고, **언제 이득인지의 조건**을 얻었다.

## 한계 · 다음에 해볼 것

- 데이터가 34개로 너무 작아 둘 다 암기 → 인코딩 효과가 학습 난이도로 드러날 여지가 좁았다.
- 논문 조건에 더 가깝게 하려면: (a) **지속 길이 범위를 넓힌** 장단(느린 중모리 등)으로 어휘 폭발 재현,
  (b) **2성부 이상 heterophony**(장구+북)로 정렬 문제 도입, (c) 데이터를 늘려 암기→일반화 구간 확보.
- 이 세 조건 중 하나라도 넣으면 위치 인코딩 이득이 살아나는지 — 그게 이 실험의 자연스러운 후속.

## 재현 방법

```bash
cd janggan && source .venv/bin/activate
python data/build_dataset.py   # 데이터 34개
python encode.py               # 왕복 검증 + 어휘 비교 (관찰①)
python train.py                # v1/v2 동일조건 학습 (관찰②의 loss)
python compare.py              # 생성 + 구조 유효성 (관찰③)
```
