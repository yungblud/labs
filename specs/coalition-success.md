# 기술공화국 기획은 성공하나 — "붕괴 ≠ 실패"를 정책 포획으로 재는 후속 실험

> 선행: `specs/coalition-fracture.md` (연합이 **언제·어떻게 깨지나** 는 이미 답함 — `coalition/NOTES.md`).
> 이 실험이 답할 다른 질문: **"무너지기 *전에* 기술공화국을 제도로 포획할 수 있었나."**
> 착상: 선행 노트 100–103줄 —
> *"지금 모델은 붕괴 시점만 본다. 고현저성 창 동안 얼마나 챙기는지를 붙이면
> 붕괴가 '성공 후 해산'인지 '실패 후 와해'인지 판별할 수 있다."*
>
> **주의: 선행과 동일하게 이론 재현용 장난감. 현실 정치 예측 아님.** 목표는 생성 정확도가
> 아니라 아래 관찰(리서치 노트)이 나오는 것. 이상점·궤적은 손으로 박은 값이다.

## 목표

선행 실험은 "적이 식으면 반드시 깨진다(100% 구조적 필연)"까지 증명했다. 그러면 남는 질문은
**붕괴 여부가 아니라 붕괴 *전에 무엇을 잠갔나***다.

**성공 = 정책 포획(policy capture).** 이념공간에 "기술공화국 목표점"을 박고, 연합이 **winning
(집권 가능 크기)인 동안** 연합중심 `center`가 그 목표점에 얼마나·얼마나 오래 근접했는지를
시간에 걸쳐 적분한다. 누적 포획량이 임계 θ 를 넘으면 **기획 성공.**

**확률 = 2요인 몬테카를로.** 결정론 모델을 확률화하는 두 노브:
- **창길이(window)** — 위협현저성 `s(t)`를 확률적 궤적으로(고점→감쇠, 감쇠속도·타이밍·충격 랜덤).
  적이 얼마나 오래 사는가 = 집권 창이 얼마나 긴가.
- **구조(structure)** — 마찰 `w`·이상점·stake 에 노이즈(선행 `sensitivity.py`의 `jitter` 재사용).
  이념거리 배치가 애초에 얼마나 험한가.

두 노브를 각각/함께 흔들어 **P(성공)** 을 재고, **분산분해**로 묻는다:
> 성공확률을 지배하는 건 **창길이(적이 빨리 식어서 실패)** 인가 **구조(이념거리가 커서 실패)** 인가.

```
① 이념공간에 기술공화국 목표점 T* 를 정의한다               ← 번역(모델링) 지점
② 확률적 위협궤적 s(t): 고점 → 랜덤 감쇠(+충격) 생성
③ 매 스텝 resolve_coalition → winning 이면 proximity(center,T*) 누적
④ 누적 포획 ≥ θ 이면 그 run 은 성공. MC 로 P(성공) 추정
⑤ 궤적-only / 파라미터-only / 둘다 로 분산분해                ← 실험 지점
⑥ "붕괴는 성공 후 해산이었나 실패 후 와해였나 / 무엇이 성공을 지배하나" 노트   ← 결론 지점
```

**성공 판정(이 실험의):** ⑥의 노트가 나오면 성공. P(성공)의 *절대 수치*는 목표 아님(손 배치의
함수라 반증 불가) — 목표는 **"창길이 대 구조 중 무엇이 성공을 가르나"의 구조적 관찰.**

## 모델 설계 — 무엇을 추가하나 (선행 재사용 + 두 축)

선행 `model.py`는 그대로 둔다(수정 없음). `Faction`·`resolve_coalition`·`coalition_center`·
`utility` 재사용. `sensitivity.py`의 `jitter`(구조 노이즈) 재사용. **새로 붙이는 것은 셋뿐.**

### 1. 기술공화국 목표점 `T*` (이념공간의 한 점)
"기획이 성공했다면 정책이 놓일 자리." 테크우파의 **국가주의 분파(Thiel·Karp) 비전** 기준 —
중앙 기술관료·안보국가 + 시장 탈규제 + 세속. (Musk·Crypto 의 반국가 편향은 *같은 연합이지만
다른 목표* 라 T* 가 아님. 이 긴장이 아래 핵심 관찰의 씨앗.)

```
T* = (religion≈0.25, nativism≈0.60, market_dereg≈0.75, state_order≈0.90)
```
근거는 `trajectory.py` 상수 주석에 인라인. (감도용으로 T* 도 흔들 수 있게 인자화.)

### 2. 확률적 위협궤적 `s(t)` — **창길이 노브**
두 위협(`china`, `cultural`)이 각각 독립 궤적. 최소 형태:
```
s_t(t) = 1.0                       (0 ≤ t < t_peak)        # 고점 유지(적이 살아있는 창)
       = exp(−k · (t − t_peak))    (t ≥ t_peak)            # 이후 감쇠
       + (드물게) 충격: 특정 t 에서 s 를 일시 재점화
```
run 마다 샘플: `t_peak`(창 지속), `k`(감쇠속도), 충격 유무/시점. 두 위협의 파라미터를 **따로**
뽑아 선행의 비대칭이 궤적에도 살아있게 한다.

### 3. 포획 적분 + 성공 판정
```
capture = Σ_t  [winning_t] · proximity(center_t, T*) · Δt
proximity(c, T*) = 1 − ‖c − T*‖₂ / d_max          # d_max = 이념공간 대각 길이(정규화)
성공 = capture ≥ θ
```
- **핵심 게이트: `winning_t` 곱.** winning 아닌 연합은 집권력이 없어 정책을 못 박는다 →
  그 스텝의 근접은 포획으로 세지 않는다. "가깝지만 소수" ≠ 포획.
- θ 는 "충분히 오래·충분히 가깝게 잡고 있었나"의 임계. 스펙에선 기저 궤적의 포획량 분포를
  보고 중앙값 근처로 잡아 성공/실패가 갈리게 캘리브레이션(값·근거는 코드 주석).

## 실험 설계 (④⑤)

1. **기저 궤적 1개 스냅샷** — 결정론적 감쇠 s(t) 하나로 capture·성공 여부·매 스텝 members/center
   를 덤프(사람이 읽는 타임라인). 붕괴 시점 vs 포획 완료 시점 대조.
2. **2요인 몬테카를로** (각 n≈500):
   - **(W) 궤적-only**: 궤적 랜덤, 구조 고정 → P(성공). 창길이가 얼마나 성공을 흔드나.
   - **(S) 파라미터-only**: 궤적 고정(기저 감쇠), 구조 랜덤(`jitter`) → P(성공).
   - **(B) 둘다**: 궤적·구조 동시 랜덤 → P(성공) 기저 추정치.
3. **분산분해**: (W)·(S)의 성공률 분산 기여를 비교 → "창길이 지배 vs 구조 지배" 판정.
   (이진 결과라 무거운 Sobol 대신 요인별 조건부 MC 로 marginal effect 를 본다.)
4. **핵심 관찰 겨냥 — 포획의 정체**: 성공 run 에서 winning 을 오래 유지한 연합의 `center` 가
   **T\*(테크공화국)** 에 수렴했나, 아니면 **큰 mass 의 MAGA 쪽**으로 끌려갔나.
   가설: winning 유지 ⇒ MAGA(최대 mass) 잔류 필요 ⇒ center 가 MAGA 로 당겨짐 ⇒ **테크공화국
   target 포획엔 구조적 천장.** → *"연합이 성공적으로 오래 갈수록, 그 성공은 테크우파의 것이
   아닐 수 있다"* 가 이 실험이 노릴 반전.

## 리서치 노트가 답할 질문 (⑥ — 진짜 성공 기준)

- 기저 궤적에서 붕괴는 **성공 후 해산**이었나 **실패 후 와해**였나? (포획 완료 시점 vs 붕괴 시점)
- P(기획 성공) 기저 추정치와, 그걸 지배하는 노브는 **창길이인가 구조인가?**
- 성공하는 궤적의 공통 조건은? (감쇠가 느려야? 충격 재점화가 있어야? 두 적이 비대칭 감쇠?)
- **포획의 정체:** winning 을 오래 유지한 연합의 정책은 테크공화국(T\*)인가 MAGA공화국인가 —
  "집권 유지 조건"과 "테크 비전"이 충돌하나?
