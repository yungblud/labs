# 미적분 이론 한 바퀴 — "정의에서 공식까지 직접 유도하는" 학습 체험

> 미적분의 두 축(미분=순간 변화율, 적분=쌓인 넓이)을 **공식으로 외우지 않고**,
> 극한이라는 하나의 아이디어에서 **손으로 유도해 내려가며** 이해한다.
> **이번 실험은 프로그래밍보다 이론 위주.** 코드는 유도한 결과를 살짝 검산하는 보조일 뿐.
> **목적은 '정답 계산'이 아니라 "왜 이 공식이 이렇게 생겼는가"를 스스로 설명할 수 있게 되는 것.**

---

## 목표

미적분 교과서가 "외워"라고 던지는 공식들을, **극한이라는 뿌리 하나에서 직접 유도**해 한 바퀴 돈다.
답을 맞히는 게 아니라, **각 공식을 "왜 그런지" 내 말로 설명·유도**할 수 있게 되는 게 성공 기준.

```
① 극한이 뭔지 정의로 이해한다 (0/0 을 피하는 아이디어)          ← 뿌리
② 미분을 극한의 정의로 세운다 → x² 를 직접 유도해 2x 를 얻는다   ← 정의에서 공식
③ 거듭제곱·상수·합 규칙이 왜 성립하는지 유도한다                 ← 규칙의 근거
④ 적분을 "넓이의 극한(리만 합)"으로 세운다                       ← 정의에서 공식
⑤ 미분과 적분이 서로 역이라는 기본정리를 이해한다                ← 두 축의 연결
⑥ "무엇을 이해했나 / 어디서 막혔나" 를 노트로 남긴다             ← 성공 판정
```

**성공 판정:** ⑥의 이론 노트가 나오면 성공. "얼마나 정밀하냐"는 부차적.
핵심은 **각 공식을 정의에서부터 손으로 유도한 흔적**이 남는 것.

---

## 성격 — 이 실험에서 무엇이 "성공"인가

- **이 실험은 이론 위주.** 프로그래밍은 최소화 — 코드는 "손유도가 맞았나" 확인하는 곁가지.
- **정밀한 수치해석기 만들기 ≠ 성공.** 성공은 극한·미분·적분의 **논리 사슬을 스스로 재구성**하는 것.
- 과도한 일반화 금지 — `x²`, `x³`, `2x` 같은 **손으로 검산 가능한 최소 함수**로 끝까지 간다.
- 12살에게 설명하듯: 각 유도는 **한 줄씩, 왜 그 줄이 나오는지** 말로 붙인다. 건너뛰는 줄 없음.

## 이론 설계 — 무엇을 어떤 순서로 유도하나

### 파트 1 — 극한 (모든 것의 뿌리)
- "순간의 기울기"는 왜 `0/0` 문제에 부딪히나 → **극한**이 그걸 어떻게 피하나.
- `h → 0` 의 의미: "0 을 대입"이 아니라 "0 에 무한히 가까워질 때 값이 향하는 곳".
- 손검산: `(4+h) → 4` 처럼 간단한 수렴을 표로 한 번만 확인(코드 보조).

### 파트 2 — 미분: 정의에서 공식 유도
- 미분계수의 정의:
  ```
  f'(x) = lim_{h→0} [ f(x+h) − f(x) ] / h
  ```
- `f(x)=x²` 를 대입해 **한 줄씩 전개** → `(x²+2xh+h²−x²)/h → 2x+h → 2x`.
- `f(x)=x³` 도 같은 방식으로 직접 유도 → `3x²` (자기 검산).

### 파트 3 — 미분 규칙이 왜 성립하나
- **거듭제곱 규칙** `xⁿ → n·xⁿ⁻¹` — 유도의 핵심(이항전개에서 왜 그 항만 남나).
- **상수배·합 규칙** — 정의에 넣으면 왜 자연스레 분리되는지.
- 규칙을 "외운다"가 아니라 "정의에서 떨어진다"로 이해.

### 파트 4 — 적분: 넓이의 극한
- 곡선 아래 넓이를 왜 막대기 합(리만 합)으로 근사하나, `n→∞` 극한이 왜 참 넓이인가.
- 부정적분 `∫xⁿ dx = xⁿ⁺¹/(n+1) + C` 가 미분의 역연산임을 규칙으로 이해(+C 의 의미).
- 정적분 `∫₀³ 2x dx = 9` 를 삼각형 넓이 `½·3·6=9` 로 손검산.

### 파트 5 — 기본정리 (두 축의 연결)
- "넓이를 x 로 미분하면 원래 함수가 나온다"는 직관을 그림/말로 이해.
- `2x --적분--> x² --미분--> 2x` 왕복이 왜 성립하는지 정의 수준에서 설명.
